A* Maze Solver

As my final project of the cross-listed mechanical engineering/computer science class, ME133: Autonomy, I implemented the A* graph-search algorithm in Python. It is an extension of Djisktra’s algorithm that uses a guiding heuristic function to ideally attain the result faster than Djisktra’s. The core logic of my code is in one of the pictures below. While there are libraries out there for implementing graphs in Python, I wanted to only use built-in data structures. I ended up using a dictionary that mapped the node coordinates to an array storing the distance from start node, heuristic value, total value, and the parent node coordinates. Once I worked out the data structures and the helper functions (calculate heuristic, find list of all valid neighbor nodes) it was simple to turn the algorithm outline into actual code. I also implemented the ability to read in maps from CSV files and visualize the solutions using matplotlib.

import matplotlib as mpl
import matplotlib.pyplot as plt
from matplotlib import colors
import numpy as np
import csv
import copy

# User Defined Variables
# Note: Program assumes start and goal are within bounds of the map and that the map is
# rectangular (ie each sublist in the list has the same length)
# Map format should be csv with 1s as obstacles, 0s as free space
#start, goal, grid = (2,5), (9,5), list(csv.reader(open('C:\\Users\\Kanan\\Desktop\\Final Project\\map1.csv')))
start, goal, grid = (1,1), (1,98), list(csv.reader(open('F:\\Autonomy\\map3_zeroes.csv')))
grid = [[int(string) for string in inner] for inner in grid]

# Function for returning list of neighbor indices while ignoring non-existent ones
X = len(grid)-1
Y = len(grid[0])-1
neighbors = lambda x, y : [(x2, y2) for x2 in range(x-1, x+2)
                                   for y2 in range(y-1, y+2)
                                   if (-1 < x <= X and
                                       -1 < y <= Y and
                                       (x != x2 or y != y2) and
                                       (0 <= x2 <= X) and
                                       (0 <= y2 <= Y))]

# Straight-line distance, multiplied by 10 and rounded to the nearest integer for simplicity
def distance(coord1, coord2):
    return round(((coord1[0] - coord2[0])**2 + (coord1[1] - coord2[1])**2)**(1/2)*10)
# Apply distance formula to goal and given coordinate
def heuristic(coord):
    return distance(coord,goal)

# Initiate open list (with start coordinate already inside) and closed list            
opened = [start]
closed = []
# Using a dictionary as the data structure for storing node information
# Format - (i,j):[g, h, f, (k,l)] where i,j are node coordinates, g, h, f are the function values, and k,l are the parent coordinates
nodes = {start:[0,heuristic(start),0+heuristic(start),start]}

# Loop until open list is empty
while len(opened) > 0:
    minf = float("inf")
    # Find open node with mininum value and set to current
    for n in opened:
        if nodes[n][2] < minf:
            minf = nodes[n][2]
            current = n
    # Switch current node to closed list
    opened.remove(current)
    closed.append(current)
    # If current is goal then solution was found
    if current == goal:
        break
    # Investigate every open or unexplored, and traversable neighbor
    for n in neighbors(current[0],current[1]):
        g = nodes[current][0] + distance(current,n)
        if (n in closed) or (grid[n[0]][n[1]] == 1):
            pass
        # If neighbor is unexplored or has a suboptimal g-cost set current as parent, and update g and f
        elif (n not in opened) or (g < nodes[n][0]):
            nodes[n] = [g, heuristic(n),g+heuristic(n),current]
            # Add the previously unexplored neighbor to open list
            if (n not in opened):
                opened.append(n)

# If map was unsolved skip the path retrieva; and visualization parts of the script
if current != goal:
    print("Failure, map is unsolvable")
    quit()

# Retrieve the path by checking parent nodes, working backward from the goal
path = [goal]
i = 0                
while True:
    parent = nodes[path[i]][3]
    path.append(parent)
    i=i+1
    if parent == start:
        break
path.reverse()

# Visualization part of the program:
# Copy array for future modification
cgrid = np.array(grid)
# Fill array with 2, 3, 4 for opened, closed, or nodes lying on the path respectively
# Otherwise leave 0 or 1 which still represent free (unexplored) and occupied spaces respectively
for i in range(len(grid)):
    for j in range(len(grid[i])):
        if (i,j) in path:
            cgrid[i][j] = 2
#        elif (i,j) in opened:
#            cgrid[i][j] = 3
#        elif (i,j) in closed:
#            cgrid[i][j] = 2
# Map colors to the different values, effectively creating the following mapping
# Free (unexplored), Occupied, Opened, Closed, Path: 0, 1, 2, 3, 4
cmap = colors.ListedColormap(['white','black','green'])
plt.imshow(cgrid,interpolation='nearest',cmap=cmap)
